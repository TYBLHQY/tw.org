# 最佳实践

如果你已经安装了 Taskwarrior 并浏览了介绍和一些教程，你可能想知道如何开始使用一些功能来帮助组织你的工作。
这是一个简单的教程，不打算完整或针对特定方法，只是一个开始，让你思考你的任务列表，以及你可能更好地依赖它的方式。

默认报告（当你只输入 `task` 时运行的报告）是 `next` 报告。
这是一个任务按紧迫性排序的报告，最紧迫的在顶部。
报告在一页后切断，所以它实际上只是最紧迫任务的列表。
按照一些提示，`next` 报告可以是你最有价值的资源。
以下是使 `next` 报告对你有利的提示。

## 提示

你会注意到这些提示都是关于为任务提供更多上下文和元数据。
如果你只需要一个简单的购物清单，那就回到纸笔，你会更满意。
但如果你想要一些复杂性和操纵和查看数据的能力...

- 捕捉所有你能的任务和细节。
  将信息从你的脑子里写到列表上可以减少你需要记住的细节数量，减少你可能忘记的东西。

- 尽可能为你的任务分配一个项目：
  
  ```
  task ID modify project:Home
  ```

- 在适当的情况下为重要任务分配截止日期：
  
  ```
  task ID modify due:31st
  ```

  不过不要过度，因为延迟可能会迫使你花太多时间重新组织一切。

- 当你开始处理一个任务时，标记它为已开始：
  
  ```
  task ID start
  ```

  这对周末之后是一个很好的提醒，你应该继续在周五做什么。

- 如果你知道任务的优先级：
  
  ```
  task ID modify priority:M
  ```

  但不要陷入经常改变优先级的陷阱，因为你会浪费很多时间。

- 为任务添加有用的标签：
  
  ```
  task ID modify +problem +house
  ```

  这对筛选非常有用。

- 添加特殊标签 +next 到任务，以提升其紧迫性：
  
  ```
  task ID modify +next
  ```

- 在适当的情况下表示依赖关系，因为阻止任务和被阻止任务的紧迫性有很大差异：
  
  ```
  task ID modify depends:OTHER_ID
  ```

- 尽量不要有大的、长期的任务坐在你的列表上永远。
  完成任务可能非常令人满意和激励，所以创建更多但更小的任务。
  不要有"学习日语"的任务，而是有"完成第 1 章"的任务，它更容易实现，你可以更容易地看到进度，这可能是鼓舞人心的。

## 如何帮助

`next` 报告按紧迫性排序。
紧迫性只是一个数字，但一个使用考虑任务许多方面的多项式计算的数字。
这意味着你为任务提供的信息越多，`next` 报告变得越准确，它越接近你自己的紧迫性概念。

如果你遵循上面的提示，你的 `next` 报告输出应该开始变得有用。
此外，通过修改紧迫性系数，你可以使 `next` 报告采用你自己的观点，例如，优先级设置是否比特定项目更重要。
以下是你可以进行的一些更改：

```
task config urgency.user.tag.problem.coefficient 4.5
```

这意味着任何具有 `+problem` 标签的任务都会获得紧迫性提升。
相反，你可以使用负系数降低不重要事情的紧迫性：

```
task config urgency.user.tag.later.coefficient -6.0
```

如果你有一个更重要的项目，你可以提升整个项目：

```
task config urgency.user.project.Home.coefficient 2.9
```

假设你不同意被阻止的任务应该有减少的紧迫性。
覆盖它：

```
task config urgency.blocked.coefficient 0.0
```

零系数意味着被阻止的任务现在对紧迫性没有*影响*。

## 仔细描述

为你的任务提供良好的描述非常有帮助。
这是一个非常糟糕的任务示例：

```
task add Renovate the kitchen
```

虽然这在最高级别上可能是对你将做什么的完美描述，但它可能是一个开放式的任务，很难评估进度。
这将是一个坐在你的任务列表中一段时间的任务，并且不是很有帮助——你从中学不到什么，它在列表中的存在会变得令人沮丧。

一个更好的方法是：

```
task add project:Kitchen Select floor tiles
task add project:Kitchen Measure counter-top
task add project:Kitchen Design placement of electrical outlets
task add project:Kitchen Locate ideal placement for extractor duct
task add project:Kitchen Select and order counter-top material
task add project:Kitchen Talk to the Electrician about when the work can start
...
```

这里 `Kitchen` 现在是一个项目名称，任务代表更小的工作单元。
虽然这意味着会花更多时间来分解较大的任务，但规划很重要。

有了更小的任务，你就有机会在你的任务之间建立链接。
例如，在要求电工开始工作之前，最好规划一下电源插座的位置。
在订购材料之前，还需要测量柜台。
这些是你可以使用任务依赖关系来形式化序列的示例。

如果你想估计项目的完成情况，拥有更多任务和更多细节将改进你的估计能力。
有了任务中的足够细节，你更有可能能够估计工作。

将任务分解成更小的任务——为更精确所需的额外努力可以通过以正确的顺序在正确的时间有效执行工作来获得回报。

## 审阅你的任务

定期检查你的列表并纠正任何错误的数据，如不正确的截止日期或不再适用的优先级（由于外部因素），甚至删除不再相关的任务。
这使你的列表保持最新，更准确地反映你需要完成的工作。
准确的元数据和良好的紧迫性系数意味着 Taskwarrior 的紧迫性概念更接近你的。
这将大有帮助。

考虑安装和使用 Taskwarrior Shell ([tasksh](../../news/news.20160905.2/)) 程序，它提供了一个 `review` 命令，可以帮助你循环浏览你的任务列表并保持更新。

有人会说，在任务列表上花费尽可能少的时间意味着有更多时间做工作。
虽然这是真的，但它假设你在做*正确的*工作。
好的建议是在任务列表上花尽可能少的时间，但要足以确保它是最新的、正确的和完整的。
然后依靠工具，去完成一些工作。

## 常识

使用任务列表，经常查看它，纠正任何错误并删除不再适用的项目。
选择适合你的方法（GTD、番茄钟等）或设计你自己的——这并不复杂。
要一致。
备份你的数据。

## 未来改进

我们一直在寻找更好的方式来表示你的任务列表、更好的方式来显示它，以及更好的方式来支持有效的方法。
我们将添加以某种方式帮助某些人的功能，我们将纠正不起作用的功能。
Taskwarrior 是一个工具包，是一个全面的工具包，旨在支持人们做工作的不同方式。
你不需要每个功能，但每个人都根据他们自己的方法使用一组不同的功能。
但你使用的每个功能都会帮助你更好地处理你的列表。
