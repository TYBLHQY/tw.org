# 工作流示例

每个人使用 Taskwarrior 的方式都不同。
这里有一些工作流示例，让你思考自己的具体方法。

## Joshua

- **您在哪里使用 Taskwarrior？**
  无处不在来追踪事物。
  我将我的个人事项分开，工作项目分为两个区域，每个都在自己的 git 存储库中。
- **您将 Taskwarrior 用于什么样的工作？**
  在工作中组织项目、将其分解为小任务、记录我正在做的事情，以及记录我如何解决问题。
- **您如何同步？您使用什么样的设备？**
  我使用自定义 git 钩子，仅提交 `pending.data` 和 `completed.data`，并推送到本地私有存储库，团队也在其中提交。
  该脚本还检测已完成的任务和任务的最新版本，以防找到相同任务的多个版本。
  目前，我正在实现一个注释器来记录项目。
- **您的默认报告是什么？**
  任务 next，带有自定义筛选器和排序顺序：
  `rc.report.next.filter=status:pending,+UNBLOCKED`
  `report.next.sort=status-,start-,priority-,project+,due+`
- **您使用标准或自定义方法论吗？**
  主要是 GTD，但会尝试添加看板元素。
- **您使用扩展或钩子脚本吗？**
  自定义 git 同步脚本。
  可能的自定义 git 编辑器脚本。
- **您最依赖的功能是什么？**
  我使用启动来查看我在做什么，但我想使用它来跟踪时间，所以我需要另一种机制来使用类似"进行中"的看板。
  我使用注释来记录程序或重要的事物。
  我还使用项目、标签、已安排和截止日期来分类任务。
- **您有意不使用的功能是什么？**
  没有紧迫性。
  没有撤消（因为这是在 git 中完成的）。
- **您如何检查您的列表？**
  我通常使用 `task` 来查看进行中的内容并继续这样做。
  我的任务按项目和优先级排序，所以我会寻找需要推进的项目并检查该项目最重要的任务。
- **其他注意事项**
  我想要一个易于部署的像看板一样的项目可视化工具。

## Paul

- **您在哪里使用 Taskwarrior？**
  我在家使用 Taskwarrior。
- **您将 Taskwarrior 用于什么样的工作？**
  我主要将其用于软件开发项目，但还有一些其他项目包括培训。
  Taskwarrior 主要是使用 Taskwarrior 来追踪功能和错误而开发的。
- **您如何同步？您使用什么样的设备？**
  我使用在 VPS 上设置的 Taskserver 来同步两个客户端、笔记本电脑和桌面。
  我是使用该 Taskserver 的多个人之一。
- **您的默认报告是什么？**
  我最常使用 `next` 报告，并调整了系数以代表我对紧迫的定义。
  `next` 报告不会产生意外，所以我可以依靠它。
  我没有自己的自定义报告。
- **您使用标准或自定义方法论吗？**
  我不遵循任何标准方法论。
  我使用项目层次结构（`tw`、`tw.242`、`tw.250`、...），所以我可以在不同级别查看项目。
- **您使用扩展或钩子脚本吗？**
  我有两个钩子脚本处于活跃状态。
  一个是 on-add 钩子脚本，将 `tw-179` 转换为 `https://github.com/GothenburgBitFactory/taskwarrior/issues/179`。
  另一个也是一个 on-add 钩子，当我添加没有项目的任务时警告我。
- **您最依赖的功能是什么？**
  我很大程度上依赖 `sync`、`next` 和虚拟标签。
  我使用 `wait` 来隐藏东西。
  这是因为我更喜欢查看更短的列表。
- **您有意不使用的功能是什么？**
  我根本不使用优先级。
  我确实使用过它们，但发现我花了太多时间摆弄这些值。
  我认为优先级是一个流动的东西，变化很大，拥有这些分配的值只意味着大量的维护工作。
- **您如何检查您的列表？**
  我在 `tasksh` 中使用一个新的审核功能，它还未完成，但已经证明有效。
  这将在 2016 年初发布。
  我还没有养成良好的审核习惯，所以它发生的频率比应该的少。
- **其他注意事项**
  - 除非真的有一个硬的、真实的截止日期，否则我不使用截止日期。
  - 当有多个相关任务和明确顺序时，我使用依赖项，但没有那么多。
  - 我使用 `start` 和 `stop` 来指示我当前正在处理的内容，所以我可以记住下一天从哪里开始。
  - 我使用一些基于关键字的颜色规则，使我关心的事物组在列表中突出。
    我为 `Documentation` 和 `Performance` 关键字以及 `+bug` 标签着色。
  - 我的列表主要是我不想忘记的东西，而不是我今天需要做的事情的列表。

  我使用单独的 Taskwarrior 数据库来追踪我的日常习惯。

- **您如何同步？您使用什么样的设备？**
  目前，我只使用一台笔记本电脑，所以我不使用同步。
  尽管如此，过去我确实使用过同步。
- **您的默认报告是什么？**
- **您使用标准或自定义方法论吗？**
  我设置了 `default.project` 为收件箱，每个任务最初都会进入这里，除非我在添加时直接为其分配特定项目。
  每个任务都属于一个项目。
  项目层次结构对应于我生活的不同领域，我大量使用子项目。
  项目示例包括 `Work.Tickets`、`Home` 或 `Volunteering.Floss.Taskwarrior`。
- **您使用扩展或钩子脚本吗？**
- **您最依赖的功能是什么？**
  由于我使用项目来区分任务所属的领域，我倾向于使用标签来区分任务的类型。
  目前，我只为编程相关的活动（`+bug`、`+rfe`、`+test` 等）一致地使用这种方法，但我计划在未来扩展它。
  我尽力不草率地添加新标签。
  我仅在仔细考虑后才使用新标签。
  我同时使用截止日期来应对真实的截止日期和自我施加的截止日期。
  这可能是错误的，因为我往往会大量移动任务，我将"截止日期"视为不如其应得的重要。
  我应该为自我施加的截止日期转移到计划。
  我使用依赖项来列举需要分解为单个步骤的任何大型任务的子任务。
  由于输入依赖项需要更多工作（列举 ID），我通常在 `taskwiki` 中进行此步骤，它可以通过缩进设置依赖项。
  我没有一个任务的多个被阻止的任务，所以我的任务列表可以以依赖项树的形式显示。
- **您有意不使用的功能是什么？**
  我不太使用优先级。
  我只是从不觉得有必要。
  我不经常使用报告。
  这是因为 Taskwarrior 目前无法显示依赖项树，这与我将任务分解为小子任务的习惯相结合，经常使任务列表令人困惑。
  我想我可以尝试配置 Taskwarrior 默认排除子任务，但我现在才想到这一点。
  我还没有使用任何自定义颜色规则。
- **您如何检查您的列表？**
- **其他注意事项**
  - 当决定如何继续进行给定的项目时，我打开相应的 `taskwiki` 文件，它作为与项目相关的注释的永久动态报告。
  - 我使用通常只是隐含项目选择器的简单上下文，例如 `project:Work`。
  - 我使用 `start` 和 `stop` 来指示我当前正在处理的内容。
    我有一个小的 tmux powerline 插件，它显示当前活跃任务（在选定的上下文中）的描述，如果我设法分散自己的注意力，它可以帮助我重新专注于当前活跃的任务。
  - 我使用 `taskwiki` 生成需要在给定日期完成的任务的每日日志。
    我也使用这个文件在几行中审查我的一天。
    这使我能够快速查看我在特定日期做了什么。
  - 我的列表试图不仅是我必须记住的任务列表，而是我所做的相当准确的描述。
    我甚至倾向于为未计划但在任何给定日期实现的内容创建和记录新的完成的任务。

  我在每个具有完整、非代理 internet 连接的设备上使用 Taskwarrior，并将这些安装与网络上自托管的 Taskserver 同步。

  "每个设备"包括我的笔记本（Fedora Linux）、一个固定的基于家的计算机（CentOS）、网络上的四个服务器（CentOS）以及我的手机和平板电脑（都是 Android）。
- **您的默认报告是什么？**
- **您使用标准或自定义方法论吗？**

  我几乎所有的任务都有截止日期，尽管我无法及时完成其中一些。
  我最重要的修饰符是 `wait`。
  这是我保持待办事项列表干净的方式。
  我使用项目和子项目来组织我的任务。

  我很少使用依赖项，主要是使用时间约束来引导。

  我根本不使用优先级。

  - 我运行最少配置，选择令我高兴的主题，星期一作为 `weekstart`、`calendar.details=full` 和 `calendar.holidays=sparse`。
