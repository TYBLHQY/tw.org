# 工作机会

{{< lead >}}对于一个开源项目家族，总有工作要做。
我们需要在所有类型的领域中得到帮助，适合各种技能水平。{{< /lead >}}

这是开源志愿者工作。
报酬为零，但您将在一组需要各种不同帮助的活跃项目中大展拳脚。
也许您是一位专业人士，可以抽出一点时间为项目做出贡献。
也许您是新手，想在一个友好的社区中活跃于开源，您可以获得大量帮助、做出急需的贡献，也许还能学到一些东西。

我们相信你们很多人想加入进来，但不知道从哪里开始。
我们会指导您并教您所需的东西。
[与我们交谈](mailto:support@gothenburgbitfactory.org)。

开源：对软件有益。对简历/履历有益。对灵魂有益。

## 问题管理员

### 职责

您拥有所有项目的 Jira 数据库。

这涉及：
- 处理新问题：填写空白、要求更多详情、拒绝疯狂的建议。
- 重现问题。
- 合并重复项。
- 评估旧问题的相关性。
- 在可能的情况下分配优先级和修复版本。
- 保持标签和版本最新。
- 链接相关问题。
- 帮助版本规划。
- 就发现的模式和弱点向开发人员提建议。
- 项目发布的否决权。

### 所需技能

- 细节注意力。
- 组织。

### 时间承诺

最少每周一到两小时。
可能更多。

## 测试自动化

### 职责

你管理所有项目的测试套件。

这涉及：
- 在必要时创建新的 Python 单元测试。
- 从 Jira 中的标签创建新测试。
- 重构不好的测试。
- 组合/减少现有的重复测试。
- 提高测试覆盖率。
  Taskwarrior 过去达到了 90% 的覆盖率，其他项目我们没有时间检查。
- 提高发布质量。
- 提高测试套件速度。
- 否决项目发布。
- 就弱点领域、功能不对称向开发部门提建议。

### 所需技能

- Python。
- 注意细节。

### 时间承诺

最少每周一到两个小时。

## 声音和狂暴

每个人都喜欢实时演示，但我们有五年前的在线教程，这些教程不仅过时了，而且太长了。

### 职责

你管理视觉演示和教程。

这涉及：
- 为 YouTube 和/或 Asciinema 创建良好的功能特定演示。
- 创建有趣的新用例的演示。
- 组织演示。
- 在网站和社交媒体上展示演示。
- 为每个版本建立视觉变更日志。

### 所需技能

- 视频捕捉。
- 必要时进行旁白。

### 时间承诺

我们不确定。
演示很费力。

## 技术社区联络

### 职责

你管理与下游客户的关系。

这涉及：
- 与发行版包维护者合作将补丁拉回上游。
- 催促发行版更新软件包。
- 确保发行版正确处理开发版/alpha/beta 版本。
- 帮助我们找到 Cygwin 包维护者。
- 向发行版提供文档建议。
- 识别缺失的发行版并提出要求。
- 解决一些问题：Debian 不会使用我们的 PDF，或更新旧的 GnuTLS 库。
- 就发行版问题和改进向开发部门提建议。

### 所需技能

沟通、无所畏惧。

### 时间承诺

每周一到两个小时。

## 文档管理

### 职责

你管理所有项目的所有文档。

这涉及：
- 所有现有文档的结构和组织。
- 现有文档的重写和组合。
- 手册页和在线文档之间的一致性问题。
- 文档的视觉展示。
- 合适的示例和截图。
- 删除/替换过时的文档。
- 否决项目发布。

### 所需技能

- 清晰的沟通。
- 组织能力。

### 时间承诺

我们不确定。
起初更多，之后更少。

## 发布

网站目前是使用模板的手写 HTML。
我们希望迁移到基于 Markdown 的系统，以生成多种文档类型。

### 职责

你管理网站发布基础设施的决定。

这涉及：
- 与我们合作定义静态网站生成器的需求。
- 评估并选择静态内容生成器。
- 概念验证。
- 建立页面模板。
- 将网站迁移到 Markdown。

### 所需技能

文档管理。

### 时间承诺

起初每周需要几个小时。
